### 哈希函数

原则：一致性（同样的x产生同样的y）/高效性（计算哈希值）/均匀性

1. 把“键”转换为索引的函数
2. 不同的键通过函数产生了相同的索引，称为哈希冲突，解决冲突
空间换时间
无限的空间，时间复杂度1
空间为1，时间复杂度就是n，类似链表
索引的分布越均匀越好

#### 整形哈希函数设计

难点：大整
例：身份证号
通常做法：取模M，比如取最后四位，（取最后6为会产生分布不均匀，多的两位日日期，最多到31）
